Архитектура должна быть модульной и чистой.
Цвета: основной синий #285A71, акцентный лаймовый #A4BE4F, фоновый кремовый #FCE4C0. Все цвета прописываются строго в tailwind.config.ts как именованные переменные: primary, accent, cream. Запрещено использовать HEX-коды напрямую в компонентах.
Шрифты: для заголовков используется шрифт onder, для основного текста kiyosuna. Названия шрифтов регистрируются в конфигурации Tailwind как family-heading и family-body.
Стиль: экстремальный минимализм с акцентом на "мягкий" дизайн. Главный стандарт — максимальный border-radius (от 24px до 32px или full) для всех кнопок, карточек и контейнеров. Обязательно использование больших внутренних отступов (padding) и свободного пространства (whitespace) между блоками.
Структура папок: компоненты делятся на ui (атомарные элементы без логики) и modules (сложные функциональные блоки). Бизнес-логика и API-запросы выносятся исключительно в composables или действия (actions) в Pinia. В папке store создаются отдельные файлы для каждой сущности. Глобальные стили описываются в assets/css/main.css через директивы @layer base для установки шрифтов и цвета текста по умолчанию.
Стандарты кода: использование TypeScript для всех компонентов и сторов. Props и Emits должны быть строго типизированы. Все текстовые константы и конфигурации выносятся в отдельные файлы. Использование утилит Tailwind обязательно для всех стилей, инлайновые стили запрещены.
